Flat collection of independent client-side React components under `src/components/`, organized into three internal layers with one-way dependency direction:
- App-shell layer: `auth-provider.tsx` (React Context + localStorage-backed JWT via `@/lib/api`) and `theme-provider.tsx` (`light|dark` persisted to `cm-theme`, synced to `data-theme` on `<html>`), consumed through `useAuth()` / `useTheme()` hooks; `site-nav.tsx` composes `ThemeToggle` and `Button` from `@/components/ui/primitives`; `legal-shell.tsx` wraps legal pages in a fixed nav + aside + footer layout.
- Feature/visual layer: presentation-only components (`AnnouncementBar`, `BranchFeatures`, `HeroTerminal`, `InstallBox`, `ModelsTable`, `PixelatedCanvasDemo`) that render static/marketing content using Tailwind utilities plus local CSS Modules.
- WebGL canvas layer: `LaserFlow.jsx` (Three.js `RawShaderMaterial` beam/flare shader) and `SplashCursor.jsx` (raw WebGL2 Navier-Stokes advection/diffusion solver with ping-pong FBOs) are self-contained canvases mounted into a ref div; they manage their own GL context, FBO ping-pong buffers, resize observers, IntersectionObserver-driven pause/resume, DPR auto-scaling, and expose only props.
No cross-imports between feature or WebGL components; all state is local or provided by the two Context providers.
