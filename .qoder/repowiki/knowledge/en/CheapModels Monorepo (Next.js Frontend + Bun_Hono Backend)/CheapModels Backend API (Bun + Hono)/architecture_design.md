Flat `src/` layout with one entry point `index.ts` that constructs a single Hono app and mounts all routes inline. Two global middlewares run on every request: a CORS handler (`Access-Control-Allow-*`) and an `/api/*` guard that delegates to `requireAuth`, which decodes a hand-rolled HS256 JWT (`auth.ts`) and attaches `userId`/`email` into Hono's typed `Variables`. Route modules are co-located per domain — `auth.ts` (JWT sign/verify, password hashing, user CRUD), `keys.ts` (API key lifecycle), `providers.ts` (BYOK provider registry), `conversations.ts` (conversation/message store), `usage.ts` (analytics & cost recording) — and share one singleton database instance exported from `db.ts`, which opens `bun:sqlite` in WAL mode, enables foreign keys, and runs DDL for six tables (`users`, `api_keys`, `providers`, `usage`, `conversations`, `messages`) on import. A local `fakeModelReply` plus the SSE `/api/stream` endpoint provide a streaming chat demo without external LLM calls; the public `/api/models` catalog is intentionally unauthenticated. The process is served via `Bun.serve({ fetch: app.fetch })` and configured through `PORT`, `CORS_ORIGIN`, `BACKEND_DB`, and `JWT_SECRET` environment variables.
