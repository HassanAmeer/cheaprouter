Two independently runnable services share one git repository: `frontend_app` is a Next.js App Router site driven by root `package.json` scripts (`dev`/`build`/`start`), while `backend_service` lives under `backend/` as a standalone Bun/Hono process. The two are not compiled together — the root `tsconfig.json` explicitly excludes `backend`, and there is no monorepo tool (pnpm workspaces / Turborepo) wiring them; each child manages its own dependencies and lifecycle. Cross-service communication is HTTP-based via a typed API client on the frontend rather than shared TypeScript packages, keeping the compile graph strictly single-process per service.
