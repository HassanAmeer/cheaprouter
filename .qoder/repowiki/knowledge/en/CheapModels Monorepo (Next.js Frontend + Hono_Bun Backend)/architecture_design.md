Two independently runnable services share one git repository: `frontend_app` is a Next.js App Router site (scripts in root `package.json`) and `backend_service` lives under `backend/` as a standalone Bun/Hono process. The two are not compiled together — the root `tsconfig.json` explicitly excludes `backend`, and the frontend talks to the backend via HTTP through a typed API client rather than shared TypeScript packages. There is no monorepo tool (pnpm workspaces / Turborepo) wiring them; each child manages its own dependencies and lifecycle.
