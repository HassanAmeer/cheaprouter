Built on Next.js App Router with `src/app/layout.tsx` as the single root layout that composes three cross-cutting providers — `ThemeProvider`, `ToastProvider`, and `AuthProvider` — so every page inherits theme, toast, and auth context. Pages under `src/app/` are split into public marketing routes (`login`, `signup`, `pricing`, `docs`, `privacy`, `terms`) and protected dashboard routes (`dashboard/*`) plus interactive demos (`chat`, `cli`). Server-side data is accessed exclusively through the centralized `src/lib/api.ts` client, which reads `NEXT_PUBLIC_API_URL`, injects a Bearer token from `localStorage('cm_token')`, and exposes grouped methods for auth, models, keys, providers, analytics, conversations, and an SSE-based `streamChat` helper; pages call these instead of issuing raw fetches. Shared React pieces live in `src/components/` (marketing shells like `SiteNav`, `LegalShell`, `HeroTerminal`, `BranchFeatures`, `InstallBox`, plus reusable `ui/` primitives such as `SpaceButton`, `Toast`, `Primitives`, charts, and pixelated canvas) and are consumed by both public pages and dashboard views. Theme tokens are centralized in `src/config/theme.ts` and applied via the Inter font variable on the root `<html>`.
