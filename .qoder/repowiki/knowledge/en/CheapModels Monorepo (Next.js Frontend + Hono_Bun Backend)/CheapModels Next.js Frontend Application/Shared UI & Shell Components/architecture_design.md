Flat collection of independent client-side React components under `src/components/`, each self-contained with its own CSS module or `.css` import. Two layers coexist:
- App-shell layer: `auth-provider.tsx` (React Context + localStorage-backed JWT via `@/lib/api`) and `theme-provider.tsx` (`light|dark` persisted to `cm-theme`), consumed through `useAuth()` / `useTheme()` hooks; `site-nav.tsx` composes `ThemeToggle` and `Button` from `@/components/ui/primitives`; `legal-shell.tsx` wraps legal pages in a fixed nav + aside + footer layout.
- Feature/visual layer: presentation-only components (`AnnouncementBar`, `BranchFeatures`, `HeroTerminal`, `InstallBox`, `ModelsTable`, `PixelatedCanvasDemo`) that render static/marketing content using Tailwind utility classes plus local CSS modules.
- WebGL canvas layer: `LaserFlow.jsx` and `SplashCursor.jsx` are standalone Three.js/WebGL2 fluid-simulation canvases mounted into a ref div; they manage their own GL context, FBO ping-pong buffers, resize observers, IntersectionObserver-driven pause/resume, and DPR auto-scaling, exposing only props.
Dependency direction is one-way: shell providers → UI primitives; visual components → no shared state; WebGL components → nothing but `react` and `three`.
